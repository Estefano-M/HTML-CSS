Ao criar um novo site é bom colocar "index" antes do ".html" pq ao fazer isso:

Comportamento padrão dos servidores web: Servidores como Apache, Nginx ou mesmo o GitHub Pages costumam buscar automaticamente um arquivo chamado index.html quando um diretório é acessado. Por exemplo, se você acessa https://exemplo.com/, o servidor automaticamente carrega https://exemplo.com/index.html, se ele existir.

URLs mais limpas: Com index.html, você pode ter uma URL como https://meusite.com/ em vez de https://meusite.com/home.html ou https://meusite.com/inicio.html.

Organização: Em projetos maiores, manter um index.html como ponto de entrada facilita a navegação e o entendimento da estrutura do site.
meusite/ ├── index.html ← página inicial ├── sobre.html ├── contato.html └── css/

Conclusão: Você não precisa usar "index" no nome, mas se estiver criando a página principal do seu site ou pasta, usar index.html é uma prática recomendada.
